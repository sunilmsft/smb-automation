🛠️ Personal Laptop Setup Guide

Continue the SMB Automation project from your personal machine

Step 1 — Install Prerequisites

You need three things. All free except Copilot (optional paid plan).

1️⃣ Visual Studio Code Free

  • Download from code.visualstudio.com
  • Install with default settings
  • Works on Windows, Mac, or Linux

2️⃣ Git Free

  • Download from git-scm.com
  • During install, accept all defaults
  • After install, open a terminal and set your identity:
git config --global user.name "Your Name"
git config --global user.email "your-email@example.com"

3️⃣ GitHub Copilot Extension In VS Code

  • Open VS Code
  • Go to Extensions (Ctrl+Shift+X)
  • Search for "GitHub Copilot"
  • Click Install (it will also install "GitHub Copilot Chat")
  • Sign in with your GitHub account when prompted

Step 3 — Open the Project

Pick whichever option is easier for you:

Option A — OneDrive (Easiest)
Option B — Git Clone

Option A: OneDrive Recommended

The project folder is already on your personal OneDrive, so it syncs automatically:

1

Sign into OneDrive on your personal laptop (if not already)

2

Wait for sync to complete

3

Open VS Code

4

File → Open Folder → navigate to:

OneDrive\GitHub Copilot Fun Projects\Tim and Sunil building Stuff!
5

Done — all files including git history are already there

Option B: Git Clone

If OneDrive isn't set up, clone fresh from GitHub:

cd ~\Documents
git clone https://github.com/sunilmsft/smb-automation.git
cd smb-automation
code .
ℹ️ If this is a fresh clone, you'll need to authenticate with GitHub. Install GitHub CLI and run gh auth login.

Step 4 — Verify Everything Works

✅ Check the file tree

You should see these files:

  • index.html — the main dashboard (~2,000 lines)
  • context.md — project context
  • memory.md — conversation/decision log
  • .github/copilot-instructions.md — Copilot reads this automatically

✅ Test Copilot Chat

Press Ctrl+Shift+I (or click the Copilot icon in the sidebar)

Ask it: "What is this project about?"

It should know the answer — because it reads .github/copilot-instructions.md automatically.

✅ Test Git

Open the terminal in VS Code (Ctrl+`) and run:

git status
git log --oneline -5

You should see recent commits.

Step 5 — Push Changes to GitHub Pages

When you make edits and want them to go live:

git add -A
git commit -m "Your commit message here"
git push
⚠️ git push may show what looks like an error (exit code 1) — this is normal. It's stderr output, but the push still succeeds. Check the live site to confirm.

What Carries Over vs. What Doesn't

ItemCarries Over?How
All code (index.html, etc.)✅ YesOneDrive sync or git clone
Project context for Copilot✅ Yes.github/copilot-instructions.md — auto-read
Decision history✅ Yesmemory.md and context.md
Git history✅ YesFull commit history in the repo
Live site✅ YesGitHub Pages at the same URL
Chat conversation history❌ NoStored locally on work laptop
Copilot memory/preferences❌ NoBut copilot-instructions.md replaces most of this
💡 Bottom line: The only thing you lose is the scrollback of past chat conversations. All decisions, code, and context are preserved in the repo. Copilot picks up the project context automatically.
